# V3.2 — 2 Agents, 2 Robots (2 Ego Cams, No Overhead)

## Setup
- **Camera:** 2 ego cameras only (no global overhead view)
- **Agent:** 2 agents, fully decentralized
- **Robot:** 2× SO-ARM100 (6-DOF)

## Architecture

```
Robot 1 ego ──→ Encoder1 ──→ z1
Robot 2 ego ──→ Encoder2 ──→ z2
                                   
z1' = z1 + CrossAttn(Q=z1, K=z2, V=z2)   # infer other's state from partial view
z2' = z2 + CrossAttn(Q=z2, K=z1, V=z1)
                                   
z1' ──→ Predictor1 ──→ MDN-GMM ──→ action1
z2' ──→ Predictor2 ──→ MDN-GMM ──→ action2
```

No overhead camera = each robot has partial observation only.
Cross-attention compensates: each agent infers the other's state from its ego view.

## Key Challenges
1. **Partial observability:** No global view → agents must infer world from ego cams
2. **Aliasing:** Robot 2 looks same from robot 1's perspective at different positions → cross-attn ambiguity
3. **Coordination:** Decentralized, no shared plan → must predict other's actions via MDN-GMM

## How CLIP Helps
CLIP joint embedding ensures:
- `z1` (robot 1's view of robot 2) is in same space as `z2` (robot 2's view of itself)
- Comparison and cross-attention remain meaningful despite different viewpoints
- Shared semantic space prevents representation drift between agents
